Message

消息处理 Message Handling

围绕“知道客户在说什么，并快速回到可发送状态”。 Built around understanding the buyer and getting back to a send-ready draft fast.

GIF 01

消息工作流 Message Flow

读取对话、中文理解、建议回复、人工发送。 Read thread, understand intent, draft reply, send manually.

打开对话后，插件读取历史上下文与待回复内容，再给出可修改的建议回复。 After a thread is opened, the extension reads context and pending buyer messages, then generates an editable draft.

  • 只建议，不自动发送。 Draft only. No auto-send.
  • 支持中文补充意见后重生成。 Rewrite again from Chinese feedback.
  • 优先处理待回复消息，不丢历史上下文。 Prioritize pending messages without losing context.
Orders

订单同步 Order Sync

围绕“当前页订单是否已经入库”。 Built around whether the current page order is already in the system.

GIF 02

订单识别 Order Check

当前页订单、新订单数量、已入库数量。 Current page count, new orders, and already-synced orders.

打开订单页后，插件抓取当前页数据，并把“新订单”和“已入库”分开显示。 Once the order page is open, the extension reads the current page and separates new orders from already-synced ones.

  • 按当前店铺隔离数据。 Store-level data isolation.
  • 同步前先做订单号校验。 Check order numbers before sync.
  • 更适合进入飞书与后台工作流。 Ready for Feishu and back-office workflow.
Pricing

定价工具 Pricing Tool

围绕“竞对赚多少，我该怎么定”。 Built around how much the competitor makes and how you should price.

GIF 03

定价拆解 Pricing Breakdown

价格、折扣、运费、利润、汇率。 Price, discount, shipping, margin, and FX.

打开竞对 Listing 后，抓价格结构，再去算竞对利润和自己的可行定价。 Open a competitor listing, capture its pricing structure, then estimate competitor margin and your own viable price.

  • 支持逆向定价与正向定价。 Supports reverse and forward pricing.
  • 比较折扣、毛利、运费与竞争差距。 Compare discounts, margin, shipping, and competitive gap.
  • 适合多国家、多币种场景。 Fits multi-country and multi-currency setups.

FAQ

FAQ

常见问题放在这里。 Common questions live here.

谁可以使用？ Who can use it?

邀请制。默认不对外公开。 Invite only. Not publicly open by default.

消息会自动发送吗？ Will it auto-send messages?

不会。插件只生成建议回复，最终发送由人工完成。 No. It drafts suggestions only. Final send stays manual.

授权是怎么生效的？ How does activation work?

一套激活码绑定当前设备，后台按模块控制可见性和到期时间。 One activation code is tied to the current device. Feature visibility and expiry are controlled per module from the backend.

为什么没有调研模块？ Why no research module?

当前稳定插件只保留消息、订单、定价。研究与 Listing 相关能力会独立成网页工具。 The stable plugin keeps only messages, orders, and pricing. Research and listing-related work will live as separate web tools.

后续会继续更新吗？ Will it keep updating?

会。版本会持续迭代，下载入口也会放在这里。 Yes. New versions will keep shipping, and this page remains the download entry.
